# 2-9 工作台首页生成

本文件用于沉淀“工作台首页”的生成方法。工作台首页是原型系统的第一屏，作用不是堆功能，而是让客户或团队快速理解：

```text
这个原型解决什么问题
现在有哪些能力
下一步应该点哪里
后端服务是否可用
方法资产在哪里
```

## 一、本节目标

本节将 2-8 的占位页升级为正式工作台首页。

首页需要承担四个任务：

| 任务 | 说明 |
|---|---|
| 说明定位 | 说明这是 FDE Prototype Factory Web Console |
| 展示能力 | 展示客户需求分析、健康检查、Prompt 模板等入口 |
| 呈现进度 | 呈现第二阶段当前训练进展 |
| 引导操作 | 告诉用户下一步进入哪个功能页 |

## 二、首页信息架构

建议首页包含以下区域：

| 区域 | 内容 | 目的 |
|---|---|---|
| 顶部导航 | 系统名称、阶段标签、状态徽标 | 建立系统感 |
| 英雄区 | 标题、说明、主操作按钮 | 说明原型价值 |
| 能力卡片区 | 核心能力入口 | 引导用户进入功能 |
| 流程区 | 从需求到原型的步骤 | 体现方法论 |
| 阶段进度区 | 当前章节进度 | 方便学习追踪 |
| 后端状态区 | FastAPI 地址和接口说明 | 为后续联调做铺垫 |

## 三、工作台首页生成 Prompt

后续你可以把下面这段 Prompt 发给 Codex/Copilot，用于生成类似首页。

```text
请为我的 Vue3 + Vite 售前原型项目生成一个工作台首页。

业务背景：
这是一个 FDE Prototype Factory Web Console，用于训练如何基于 VSCode、Codex/Copilot、Vue3 和 FastAPI 快速生成业务原型。

页面目标：
1. 让用户一眼看懂系统定位。
2. 展示当前原型能力入口。
3. 展示“客户需求 -> 功能设计 -> 页面设计 -> API 契约 -> Vue3 原型”的方法流程。
4. 展示当前第二阶段练习进度。
5. 为后续 FastAPI 健康检查和客户需求分析页面预留入口。

技术要求：
1. 使用 Vue3 单文件组件。
2. 使用 `<script setup>` 管理页面数据。
3. 使用数组渲染能力卡片、流程步骤和进度清单。
4. 不引入 UI 组件库。
5. 样式使用普通 CSS，企业级、克制、适合售前演示。
6. 页面要支持移动端响应式。
7. 不要调用真实接口，后端状态先用静态占位。

视觉要求：
1. 白底或浅灰背景。
2. 蓝色作为主色。
3. 卡片圆角不超过 8px。
4. 信息密度适中，不要做夸张营销页。
5. 第一屏要能看到系统定位和核心能力入口。
```

## 四、首页验收清单

| 检查项 | 合格标准 |
|---|---|
| 定位是否清楚 | 标题和说明能看出系统是 AI 原型工厂 |
| 入口是否明确 | 能看到客户需求分析、健康检查、Prompt 模板等入口 |
| 方法是否呈现 | 能看到从需求到原型的流程 |
| 进度是否清楚 | 能看到第二阶段当前进展 |
| 视觉是否克制 | 适合企业级售前原型，不花哨 |
| 是否可扩展 | 后续能继续拆出组件和路由 |
