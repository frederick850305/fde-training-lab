# 2-45 页面区域生成联调修正

## 背景

在 `04 页面设计` 中点击“调用大模型生成区域”时出现 `Failed to fetch`。原因是前端默认调用 `http://127.0.0.1:8000`，但项目启动脚本 `scripts/start_api.sh` 使用的是 `8001` 端口。

## 本节改动

- 将 `PageDesignView.vue` 的默认 API 地址调整为 `http://127.0.0.1:8001`。
- 将 `LlmStepRevisionPanel.vue` 的默认 API 地址同步调整为 `http://127.0.0.1:8001`。
- 优化页面区域生成失败提示：
  - 后端未启动或端口不一致时，不再直接显示 `Failed to fetch`。
  - 明确提示启动 `bash scripts/start_api.sh`。
- 增加本地规则兜底：
  - 大模型不可用时，先根据当前页面、当前业务场景和功能点生成页面区域。
  - 生成后自动保存本地草稿，避免工作中断。

## 页面区域的用途

页面区域用于把当前选中的页面拆成可设计的界面区块，例如状态概览、列表、筛选、详情、反馈和保存区。下一步 `05 交互设计` 会基于这些页面区域继续设计字段、按钮、校验和状态。
