# 2-24 计划任务跟踪原型页

## 本节目标

本节在 2-23 的“项目总览原型页”基础上，继续生成第二个 P0 业务原型页面：计划任务跟踪页。

当前仍然不调用 DeepSeek，也不调用 FastAPI。本节使用 `prototypeMockData.js` 中的计划任务模拟数据，并复用 `DataTable.vue` 和 `StatusTag.vue`，完成一个可展示的计划任务跟踪原型页面。

## 本节价值

项目总览页面向项目经理，偏“看整体”。

计划任务跟踪页面向计划员，偏“看任务、筛状态、识别延期和阻塞”。

这两个页面组合起来后，原型就开始具备从管理总览到计划执行跟踪的基本业务闭环。

## 页面设计

`ScheduleTrackingView.vue` 包含：

1. 页面标题区
   - 页面名称
   - 当前模式提示

2. 筛选区
   - 项目
   - 区域
   - 任务状态
   - 筛选按钮

3. 状态摘要区
   - 全部任务
   - 进行中
   - 延期
   - 阻塞

4. 任务列表区
   - 任务编号
   - 任务名称
   - 区域
   - 责任人
   - 状态
   - 截止日期

5. 调整建议区
   - 基于延期和阻塞任务给出模拟建议

## 当前交互规则

- 当前页面只使用本地 mock 数据。
- 当前页面不调用后端接口。
- 当前页面提供筛选表单外观，但筛选按钮暂不触发真实查询。
- 状态展示复用 `StatusTag.vue`。
- 任务列表复用 `DataTable.vue`。

## 后续扩展方向

后续可以继续扩展：

- 实现真实筛选交互
- 增加 loading/empty/error 状态切换

## 验收方式

启动前端后检查：

```bash
cd frontend
npm run dev
```

浏览器打开页面后确认：

- 工作区切换中出现“计划任务”

- 可以看到筛选区
- 可以看到状态摘要
- 可以看到计划任务表格
