# 2-18 页面清单设计模拟工作台

## 本节目标

本节在 2-17 的“功能模块设计模拟工作台”基础上，继续完成原型生成流程的第四步：页面清单设计。

当前仍然不调用 DeepSeek，也不调用 FastAPI。本节通过前端模拟数据，把功能模块进一步转成页面清单、导航结构、页面区域和 Vue 文件建议。

## 本节价值

功能模块设计回答的是“系统需要哪些能力”。

页面清单设计回答的是“这些能力应该落在哪些页面上，每个页面包含哪些区域，以及前端应该如何组织文件”。

这一步可以帮助 FDE 把业务设计转成前端工程结构。后续让 Codex/Copilot 生成 Vue3 页面时，不再只给一句“做个项目管理页面”，而是能明确告诉 AI：生成哪个文件、页面由哪些区域组成、入口导航怎么组织。

## 页面设计

`PageDesignView.vue` 包含四类信息：

1. 页面清单
   - 页面名称
   - 页面类型
   - 优先级
   - 对应 Vue 文件

2. 当前页面详情
   - 页面目标
   - 承载功能模块
   - 页面区域

3. 导航结构
   - 一级导航
   - 默认页面
   - 入口顺序

4. 文件结构建议
   - `views`
   - `components`
   - `data`

## 当前交互规则

- 页面默认展示全部页面清单。
- 点击页面卡片后，右侧展示该页面的页面目标、承载功能和页面区域。
- 底部展示导航结构和文件结构建议。

## 后续扩展方向

后续可以继续扩展：

- 将 2-17 的功能模块结果传入页面设计页面
- 增加页面优先级筛选
- 增加“确认页面清单”按钮
- 接入 DeepSeek 生成真实页面清单
- 将页面清单继续传给页面字段与交互设计步骤

## 验收方式

启动前端后检查：

```bash
cd frontend
npm run dev
```

浏览器打开页面后确认：

- 工作区切换中出现“页面设计”
- 点击“页面设计”后进入页面清单设计模拟工作台
- 可以看到页面清单
- 点击不同页面，右侧页面详情会变化
- 底部可以看到导航结构和文件结构建议
