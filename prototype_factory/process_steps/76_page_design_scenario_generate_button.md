# 2-76 页面设计按业务场景生成页面方案

## 本次目标

在第 04 步页面设计的业务场景卡片上增加“生成”按钮：

- 点击按钮先选中对应业务场景。
- 将第 03 步功能模块映射中该场景对应的 `scenario + modules` 传给 LLM。
- 由 LLM 生成页面清单、导航结构、文件结构和当前页面基础信息。

## 已完成

- `PageDesignView.vue` 将业务场景卡片改为可点击的 `article role=tab`，避免按钮嵌套按钮。
- 每个业务场景卡片右上角新增“生成”按钮。
- 新增 `generateScenarioPageDesign(scenario)`：点击后选中该场景并调用页面方案生成逻辑。
- LLM 入参新增 `featureMapping`，明确传入上一步该场景保存的功能模块映射数据。
- 生成中显示“生成中”，完成后提示已生成页面清单和当前页面基础信息。

## 数据来源

页面方案生成输入来自第 03 步保存的 `allMappings`：

- `scenario`：当前业务场景。
- `modules`：当前业务场景下的功能模块映射。

## 后续

页面区域仍由当前页面下方“调用大模型生成”单独生成；页面设计完整成果在确认进入下一步时保存到 `page.md`。
