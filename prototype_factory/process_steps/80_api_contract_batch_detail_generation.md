# 80 API 契约批量详情生成

## 目标

优化第 06 步「API 契约」的契约详情生成方式：不再要求用户逐个接口点击生成，而是在接口列表区域统一触发批量生成，并确保中途某个接口调用失败时不会阻断整批结果。

## 本次调整

- 在 API 列表头部新增「生成契约」按钮，点击后按接口顺序逐项调用大模型生成契约详情。
- 删除当前接口详情区原有的单接口「生成契约」按钮，避免操作入口重复。
- 每个接口生成前标记为「生成中」，成功后标记为「已生成」，连续重试失败后标记为「失败」。
- 单个接口最多重试 3 次；重试时使用更紧凑的页面交互上下文，降低超时概率。
- 每完成一个接口，无论成功或失败，都会立即更新本轮 API 设计结果并写入本页 Markdown，避免长批次中断导致已生成内容丢失。
- 选中接口后仅展示该接口的契约详情；如果本地 Markdown 中没有该接口详情，则显示占位提示。

## 数据闭环

- 接口清单仍来自第 05 步「页面交互」结果。
- 契约详情按接口逐项生成，写回 `projectContext.stepResults.api.contracts`。
- 本地持久化仍通过父组件统一保存到 `prototype_factory/local_step_outputs/api.md`。
- 确认进入下一步时继续传递完整 API 契约结果给第 07 步「前端原型方案」。
