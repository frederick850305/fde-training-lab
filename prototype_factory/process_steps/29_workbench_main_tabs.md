# 2-29 工作台一级 Tab 结构重组

## 本节目标

本节在 2-28 的“原型页面统一导航优化”基础上，对前台工作台首页的信息架构进行重组。

当前仍然不调用 DeepSeek，也不调用 FastAPI，不引入新的 npm 依赖。本节只调整前端展示结构，将训练方法、练习进度和业务原型系统拆分到三个一级 Tab 中。

## 本节价值

随着第二阶段练习逐步深入，首页承载的信息越来越多：

- 原型工厂方法和能力入口
- 第二阶段练习进展
- 已经生成并持续完善的业务原型页面

如果继续把所有内容纵向堆叠在一个页面中，后续新增原型页面和练习小节时会越来越难阅读。

本节通过一级 Tab 将内容分层：

1. 原型工厂方法
2. 练习进展
3. 业务原型系统

## 页面结构

### 1. 原型工厂方法

用于展示相对稳定的方法资产和训练工作台入口：

- 原型工作台能力入口
- 从需求到原型的标准路径
- 原有工作区切换
- 需求输入、场景识别、功能设计、页面设计、交互设计、API 契约、原型建议、原型流程、需求分析等设计工作台页面

### 2. 练习进展

用于单独展示第二阶段练习进度：

- 已完成小节
- 当前小节
- 后续可扩展验收状态和产物链接

### 3. 业务原型系统

用于承载随着练习不断完善的业务前台原型：

- 项目总览
- 计划任务
- 异常闭环
- 现场调度
- 物料到货

该 Tab 继续使用 2-28 新增的 `PrototypeNav.vue` 作为业务原型内部导航。

## 当前交互规则

- 一级 Tab 控制当前展示的信息区域。
- 原有工作区切换不删除，保留在“原型工厂方法”中。
- 点击 P0 原型相关入口时，自动进入“业务原型系统”Tab。
- 进入“业务原型系统”时，如果当前没有选中 P0 页面，则默认展示“项目总览”。
- 当前仍然使用本地 mock 数据。

## 后续扩展方向

后续可以继续扩展：

- 将一级 Tab 抽成通用组件。
- 为练习进展增加产物链接和验收状态。
- 在业务原型系统中增加项目选择、角色切换和当前上下文。
- 接入 Vue Router 后，将一级 Tab 和业务页面映射为真实路由。

## 验收方式

执行前端构建：

```bash
cd frontend
npm run build
```

浏览器打开页面后确认：

- 顶部 Hero 下方出现三个一级 Tab。
- “原型工作台能力入口”和“从需求到原型的标准路径”只在“原型工厂方法”Tab 中展示。
- “第二阶段练习进展”只在“练习进展”Tab 中展示。
- “业务原型系统”Tab 中展示业务原型内部导航。
- 业务原型内部导航可以在五个 P0 页面之间切换。
- 原有工作区切换仍然存在且可用。
- 第二阶段练习进度更新为 `2-29：工作台一级 Tab 结构重组`。
