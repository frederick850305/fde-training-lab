# 67 Scenario Two Stage LLM Generation

## 目标
调整第 02 步“场景识别”的生成逻辑：从第 01 步保存进入第 02 步时，不立即调用 LLM，而是先展示用户角色和业务场景占位。用户点击“重新拆解”后，先调用 LLM 生成用户角色；用户确认角色后，再基于角色调用 LLM 生成业务场景、任务流程和页面映射。

## 修改内容
- `ScenarioIdentificationView.vue` 初始化逻辑改为 `resetScenarioPlaceholders()`，不再自动调用 LLM。
- 用户角色和业务场景为空时显示占位提示。
- “重新拆解”按钮改为调用用户角色拆分 LLM。
- 增加用户角色确认弹窗，展示 LLM 拆分出的角色、职责和场景标签。
- 点击“确认并生成业务场景”后，再调用 LLM 基于已确认角色生成业务场景、任务流程和页面映射。
- 业务场景生成接口返回结构改为 JSON 对象 `{ scenarios: [] }`，避免后端 JSON 对数组根节点解析不稳定。
- 保留 30 秒请求超时和请求取消机制。

## 验证
- 已运行 `npm run build`，构建通过。
