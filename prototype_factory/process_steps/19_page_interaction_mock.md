# 2-19 页面字段与交互设计模拟工作台

## 本节目标

本节在 2-18 的“页面清单设计模拟工作台”基础上，继续完成原型生成流程的第五步：页面字段与交互设计。

当前仍然不调用 DeepSeek，也不调用 FastAPI。本节通过前端模拟数据，把页面清单进一步细化为字段、按钮、校验规则和页面状态。

## 本节价值

页面清单设计回答的是“系统有哪些页面，每个页面有哪些区域”。

页面字段与交互设计回答的是“页面上具体有什么字段、按钮怎么触发、输入如何校验、loading/成功/失败状态如何展示”。

这一步是从“页面结构”走向“可交互原型”的关键。后续让 Codex/Copilot 生成 Vue3 页面时，如果字段、按钮、校验和状态都已经明确，生成出来的页面就更像真实系统，而不是静态展示页。

## 页面设计

`PageInteractionView.vue` 包含四类信息：

1. 页面选择
   - 项目总览页
   - 计划任务跟踪页
   - 异常问题闭环页

2. 字段设计
   - 字段名
   - 字段类型
   - 是否必填
   - 字段说明

3. 按钮与动作
   - 按钮名称
   - 触发动作
   - 结果反馈

4. 校验与状态
   - 校验规则
   - loading 状态
   - 成功状态
   - 失败状态
   - 空状态

## 当前交互规则

- 页面默认展示“项目总览页”的交互设计。
- 点击不同页面后，右侧字段、按钮、校验和状态设计会变化。
- 页面底部展示通用交互规范。

## 验收方式

```bash
cd frontend
npm run dev
```

- 工作区切换中出现“交互设计”

- 可以看到页面选择列表
