# 2-16 场景识别模拟工作台

## 本节目标

本节在 2-14 的“需求拆解模拟结果”之后，继续完成原型生成流程的第二步：场景识别。

当前仍然不调用 DeepSeek，也不调用 FastAPI。本节通过前端模拟数据，展示用户角色、业务场景、任务流程和页面映射，让原型工厂开始具备从“需求”走向“场景”的能力。

## 本节价值

需求拆解回答的是“客户想解决什么问题”。

场景识别回答的是“谁，在什么场景下，怎样使用系统完成任务”。

如果没有场景识别，后续功能模块和页面设计就容易变成泛泛而谈的功能堆砌。本节的目标是让前台先沉淀一种结构化展示方式，后续接入 DeepSeek 后，也要求模型按这个结构返回结果。

## 页面设计

`ScenarioIdentificationView.vue` 包含四类信息：

1. 用户角色
   - 角色名称
   - 关注点
   - 典型动作

2. 业务场景
   - 场景名称
   - 场景说明
   - 优先级

3. 任务流程
   - 从用户进入系统到完成任务的步骤

4. 页面映射
   - 角色
   - 场景
   - 建议页面

## 当前交互规则

- 页面默认展示模拟场景识别结果。
- 用户可以点击顶部场景按钮切换当前场景。
- 切换场景后，右侧展示该场景对应的任务流程和页面建议。

## 后续扩展方向

后续可以继续扩展：

- 将 2-14 的需求拆解结果传入场景识别页面
- 增加“重新识别场景”按钮
- 增加“确认场景并进入功能设计”按钮
- 接入 DeepSeek 生成真实场景识别结果
- 将场景识别结果继续传给功能模块设计步骤

## 验收方式

启动前端后检查：

```bash
cd frontend
npm run dev
```

浏览器打开页面后确认：

- 工作区切换中出现“场景识别”
- 点击“场景识别”后进入场景识别模拟工作台
- 可以看到用户角色卡片
- 可以看到业务场景列表
- 点击不同场景，右侧任务流程和页面映射会变化
- 第二阶段练习进度更新为 `2-16：场景识别模拟工作台`
