# 2-20 API 契约设计模拟工作台

## 本节目标

本节在 2-19 的“页面字段与交互设计模拟工作台”基础上，继续完成原型生成流程的第六步：API 契约设计。

当前仍然不调用 DeepSeek，也不调用 FastAPI。本节通过前端模拟数据，把页面字段、按钮动作和状态设计转成接口路径、请求参数、响应结构和错误码。

## 本节价值

页面字段与交互设计回答的是“页面如何操作”。

API 契约设计回答的是“前端操作应该调用哪个接口，传什么参数，拿到什么返回，失败时如何展示”。

这一步是前后端协作的核心边界。后续接入 FastAPI 时，前端页面不应该临时猜接口，而是先有稳定 API 契约，再让后端按契约实现服务。

## 页面设计

`ApiContractView.vue` 包含五类信息：

1. API 列表
   - 接口名称
   - 请求方法
   - 接口路径
   - 来源页面

2. 接口详情
   - 接口目标
   - 前端触发动作
   - 请求方法与路径

3. 请求参数
   - 参数名
   - 类型
   - 是否必填
   - 说明

4. 响应结构
   - 成功响应
   - 失败响应

5. 错误码
   - 错误码
   - 含义
   - 前端展示建议

## 当前交互规则

- 页面默认展示第一个 API 契约。
- 点击不同 API 后，右侧接口详情、参数、响应和错误码会变化。
- 页面底部展示统一响应格式约定。

## 验收方式

启动前端后检查：

```bash
cd frontend
npm run dev
```

- 工作区切换中出现“API 契约”
- 点击“API 契约”后进入 API 契约设计模拟工作台
- 可以看到 API 列表
- 点击不同 API，右侧接口详情、请求参数、响应结构和错误码会变化
- 底部可以看到统一响应格式约定
