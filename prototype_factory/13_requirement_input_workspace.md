# 2-13 需求输入工作台

## 本节目标

本节要完成 AI 原型工厂的第一个真实交互入口：让用户可以在前端页面中录入客户原始需求，并通过前端模拟生成“已接收、可进入需求拆解”的状态。

当前仍然不调用 DeepSeek，也不调用 FastAPI。本节只完成前台输入工作台和模拟交互，为后续把输入内容传递给“需求拆解”步骤做准备。

## 本节价值

在原型工厂中，客户需求输入是整个流程的起点。

如果没有一个稳定的输入工作台，后续的需求拆解、场景识别、功能设计、页面设计、API 契约设计都没有明确的数据来源。

本节完成后，系统将具备：

- 输入客户原始需求的能力
- 快速填充示例需求的能力
- 清空输入内容的能力
- 输入字数统计能力
- 模拟开始分析的交互反馈
- 提示当前阶段与下一步处理方向

## 页面设计

`RequirementInputView.vue` 页面包含两个区域：

1. 左侧输入区
   - 客户需求输入框
   - 示例需求按钮
   - 清空按钮
   - 开始分析按钮
   - 字数统计

2. 右侧状态区
   - 当前阶段提示
   - 后续流转说明
   - 模拟分析结果

## 当前交互规则

- 输入为空时，点击“开始分析”会提示必须输入客户需求。
- 点击“填入示例需求”会自动填入一段海工生产运营类示例需求。
- 点击“清空”会清空输入内容和结果状态。
- 点击“开始分析”后，页面会模拟 loading 状态。
- loading 结束后，右侧展示模拟成功结果。

## 后续扩展方向

后续小节可以继续扩展：

- 将客户需求输入内容传给需求拆解步骤
- 将输入内容保存到本地状态或后端
- 接入 FastAPI 的需求分析接口
- 接入 DeepSeek，生成真实结构化需求拆解结果
- 将需求拆解结果继续传递到场景识别、功能设计和页面设计步骤

## 验收方式

启动前端后检查：

```bash
cd frontend
npm run dev
```

浏览器打开 Vite 地址，确认页面中出现“需求输入工作台”。

验收点：

- 能看到客户需求输入框
- 点击“填入示例需求”后，输入框自动填充内容
- 字数统计随输入内容变化
- 点击“清空”后，输入框和结果状态被重置
- 输入为空时点击“开始分析”会出现校验提示
- 输入内容后点击“开始分析”，右侧先显示 loading，再显示模拟成功结果
