# 2-36 业务原型系统动态集成

## 本节目标

让“业务原型系统”不再是静态的演示，而是能够体现你在“原型工厂方法”中设计的选择：

1. **项目名称动态同步**：原型总览页显示你在方法工作台中确定的项目名称。
2. **导航菜单动态联动**：根据你在“功能模块设计”阶段确认的模块，动态调整原型系统的导航菜单。例如选了“现场调度看板”，则原型导航优先突出调度相关页面。

## 本节改动

1. `App.vue` 逻辑增强
- 定义 `activePrototypeNavItems` 计算属性。
- 业务逻辑：如果已确认功能模块，则将对应的原型页面排在最前并标记，否则显示全量菜单。
- 进度列表更新 2-35 为完成，新增 2-36。

2. `ProjectOverviewView.vue` 数据联动
- 接收 `projectContext` 作为 props。
- 将页面主标题、背景描述替换为 `projectContext` 中的真实数据。

3. `PrototypeNav.vue` 样式微调（可选）
- 确保动态生成的导航项显示正常。

## 涉及文件

- `frontend/src/App.vue`
- `frontend/src/views/ProjectOverviewView.vue`
- `prototype_factory/36_prototype_system_dynamic_integration.md`

## 验收方式

1. 在方法工作台输入自定义需求（如：海工物料管理）。
2. 一路点击 Next 到达功能设计，确认“物料到货管理”模块。
3. 切换到顶部的“业务原型系统”页签。
4. 验证：
- 项目总览页显示的项目名称是你输入的名字。
- 原型内部导航中，“物料到货”排在显眼位置或被过滤出来（取决于具体实现）。
