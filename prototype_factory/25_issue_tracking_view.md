# 2-25 异常问题闭环原型页

## 本节目标

本节在 2-24 的“计划任务跟踪原型页”基础上，继续生成第三个 P0 业务原型页面：异常问题闭环页。

当前仍然不调用 DeepSeek，也不调用 FastAPI。本节使用 `prototypeMockData.js` 中的异常问题模拟数据，并复用 `DataTable.vue` 和 `StatusTag.vue`，完成一个可展示、可模拟新增异常的异常问题闭环页面。

## 本节价值

项目总览页回答“整体情况怎么样”。

计划任务页回答“计划任务执行到哪了”。

异常问题闭环页回答“发现问题后如何登记、分派、跟踪和关闭”。

这三个 P0 页面组合起来，就形成了一个最小的生产运营原型闭环：

```text
总览发现风险
  -> 计划识别延期/阻塞
  -> 异常登记与闭环跟踪
```

## 页面设计

`IssueTrackingView.vue` 包含：

1. 页面标题区
   - 页面名称
   - 当前模式提示

2. 异常新增表单
   - 异常标题
   - 责任人
   - 处理时限
   - 异常说明
   - 提交按钮

3. 异常状态摘要
   - 全部异常
   - 待处理
   - 处理中
   - 已关闭

4. 异常问题列表
   - 异常 ID
   - 异常标题
   - 责任人
   - 状态
   - 截止日期

5. 闭环规则提示
   - 发现问题
   - 分派责任人
   - 跟踪处理
   - 关闭问题

## 当前交互规则

- 当前页面使用本地 mock 数据。
- 当前页面不调用后端接口。
- 提交异常时只做前端内存级模拟新增。
- 异常标题、责任人、处理时限为必填。
- 状态展示复用 `StatusTag.vue`。
- 异常列表复用 `DataTable.vue`。

## 后续扩展方向

后续可以继续扩展：

- 增加编辑异常功能
- 增加更新状态功能
- 增加 loading/empty/error 状态
- 将模拟新增替换为 `POST /api/issues`
- 将异常列表替换为后端接口返回

## 验收方式

启动前端后检查：

```bash
cd frontend
npm run dev
```

浏览器打开页面后确认：

- 工作区切换中出现“异常闭环”
- 点击“异常闭环”后进入异常问题闭环原型页
- 可以看到异常新增表单
- 可以看到异常状态摘要
- 可以看到异常问题表格
- 输入必填字段后点击“提交异常”，列表会新增一条异常
- 第二阶段练习进度更新为 `2-25：异常问题闭环原型页`
