# 2-44 页面设计 Tab 化与当前页面区域生成

## 目标

根据页面设计工作台最新验收意见，压缩“上一步结果”的展示密度，让业务场景横向展开成 4 个小 tab；当前 tab 下只展示该场景对应的页面清单、文件结构和导航结构；页面区域要与左侧选中的当前页面联动，并支持调用大模型按当前页面真实生成。

## 本节改动

- `PageDesignView.vue`
  - 将全部业务场景改为横向 4 个 tab，仅展示优先级、场景名称和角色。
  - 将当前场景的页面清单、文件结构建议、导航结构放在 tab 下方展示。
  - 将大模型修改面板移动到页面底部。
  - 页面详情区继续与左侧页面清单联动。
  - 页面区域新增“调用大模型生成区域”，调用 `/llm/revise-step` 后只更新当前选中页面的 `sections`。
  - 页面基础字段、承载功能和区域编辑后会标记为“待保存”。

## 验证

- `npm run build` 通过。
