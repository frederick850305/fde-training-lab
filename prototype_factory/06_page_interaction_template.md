# 2-6 页面字段与交互设计模板

本文件用于把 2-5 的页面清单继续细化到字段、按钮、校验、页面状态和交互反馈。

核心原则：

```text
页面设计不能只写“有一个表单”，而要写清楚表单里有什么字段、按钮做什么、失败时如何提示。
```

## 一、本节目标

完成 2-5 后，我们已经知道：

```text
有哪些页面
每个页面服务什么角色
每个页面对应什么场景
每个页面大致包含哪些区域
```

2-6 要进一步回答：

```text
每个页面有哪些字段？
每个字段是否必填？
按钮点击后发生什么？
哪些地方需要校验？
loading、成功、失败、空数据状态如何展示？
接口返回后页面如何变化？
```

这一步的输出会直接用于：

```text
Vue3 页面生成 Prompt
表单组件设计
API 调用逻辑
错误提示组件
前端测试和人工验收
```

## 二、为什么要先设计交互再生成代码

如果直接让 AI 生成页面，常见问题是：

```text
字段不完整
按钮含义不清
错误提示很随意
loading 状态缺失
成功后不知道展示什么
失败后只显示“失败”
```

所以在生成 Vue3 代码前，必须先明确：

```text
用户输入什么
系统校验什么
点击按钮发生什么
接口返回后展示什么
异常时用户怎么继续操作
```

这就是页面交互设计的价值。

## 三、页面字段设计模板

字段设计建议固定输出以下表格。

| 页面名称 | 字段名称 | 字段标识 | 字段类型 | 是否必填 | 默认值 | 校验规则 | 说明 |
|---|---|---|---|---|---|---|---|
| 客户需求分析页 | 需求文件路径 | input_path | 文本输入 | 是 | `data/customer-requirement.md` | 不能为空 | 后端读取的需求文件路径 |

字段类型建议使用这些稳定表达：

| 字段类型 | 适用场景 |
|---|---|
| 文本输入 | 路径、名称、关键词 |
| 多行文本 | Prompt、说明、备注 |
| 开关 | 是否启用 AI、是否保存结果 |
| 下拉选择 | 类型、状态、优先级 |
| 日期选择 | 开始时间、结束时间 |
| 数字输入 | 数量、阈值、权重 |
| 文件上传 | 后续扩展文档上传 |

## 四、页面按钮设计模板

按钮设计不要只写“提交按钮”，要写清楚动作和状态。

| 页面名称 | 按钮名称 | 按钮类型 | 触发动作 | 前置条件 | 点击后状态 | 成功后反馈 | 失败后反馈 |
|---|---|---|---|---|---|---|---|
| 客户需求分析页 | 生成报告 | 主按钮 | 调用 `/requirement/summary` | 必填字段已通过校验 | loading | 展示输出路径 | 展示错误信息 |

按钮类型建议：

| 类型 | 说明 |
|---|---|
| 主按钮 | 页面最核心动作 |
| 次按钮 | 辅助动作，如重置、返回 |
| 文本按钮 | 轻量操作，如复制、查看详情 |
| 危险按钮 | 删除、清空等高风险操作 |

## 五、页面状态设计模板

核心业务页面至少要有 5 类状态。

| 页面名称 | 状态 | 触发条件 | 页面表现 | 用户可操作项 |
|---|---|---|---|---|
| 客户需求分析页 | 默认 | 页面首次打开 | 展示默认表单和空结果区 | 填写参数、点击生成 |
| 客户需求分析页 | 校验失败 | 必填字段为空 | 字段下方显示提示 | 修改字段 |
| 客户需求分析页 | loading | 正在调用接口 | 按钮禁用、显示处理中 | 等待 |
| 客户需求分析页 | 成功 | API 返回成功 | 显示成功提示和输出路径 | 复制路径、再次生成 |
| 客户需求分析页 | 失败 | API 返回失败或网络异常 | 显示错误说明和错误码 | 修改参数、重试 |

状态设计的关键是：

```text
每个状态都要告诉用户：现在发生了什么、接下来能做什么。
```

## 六、表单校验设计模板

校验分为前端校验和后端校验。

| 字段 | 前端校验 | 后端校验 | 错误提示 |
|---|---|---|---|
| input_path | 不能为空 | 文件必须存在 | 请输入需求文件路径 / 文件不存在 |
| output_path | 不能为空 | 输出目录可写 | 请输入报告输出路径 |
| enable_ai_advice | 无 | 如果开启，需要 API Key | AI 建议未配置，请检查 DeepSeek API Key |

前端校验的作用：

```text
减少明显错误请求
提升用户体验
```

后端校验的作用：

```text
保证真实业务安全
处理前端无法判断的问题
```

## 七、接口响应适配模板

页面必须知道接口返回后怎么展示。

成功响应：

```json
{
  "success": true,
  "message": "客户需求结构化分析报告生成成功",
  "data": {
    "output_path": "output/api_requirement_summary.md"
  }
}
```

页面展示建议：

| 响应字段 | 页面展示 |
|---|---|
| `success` | 判断展示成功或失败状态 |
| `message` | 作为主提示文案 |
| `data.output_path` | 展示生成报告路径 |

失败响应：

```json
{
  "success": false,
  "message": "输入文件不存在",
  "error_code": "FILE_NOT_FOUND"
}
```

页面展示建议：

| 响应字段 | 页面展示 |
|---|---|
| `success` | 展示失败状态 |
| `message` | 展示错误说明 |
| `error_code` | 展示错误码，便于排查 |

## 十、示例：客户需求分析页

### 1. 页面字段设计

| 页面名称 | 字段名称 | 字段标识 | 字段类型 | 是否必填 | 默认值 | 校验规则 | 说明 |
|---|---|---|---|---|---|---|---|
| 客户需求分析页 | 需求文件路径 | input_path | 文本输入 | 是 | `data/customer-requirement.md` | 不能为空 | 后端读取的客户需求 Markdown |
| 客户需求分析页 | 报告输出路径 | output_path | 文本输入 | 是 | `output/api_requirement_summary.md` | 不能为空 | 生成报告保存路径 |
| 客户需求分析页 | 启用 AI 建议 | enable_ai_advice | 开关 | 否 | false | 无 | 是否调用 DeepSeek 生成实施建议 |

### 2. 页面按钮设计

| 页面名称 | 按钮名称 | 按钮类型 | 触发动作 | 前置条件 | 点击后状态 | 成功后反馈 | 失败后反馈 |
|---|---|---|---|---|---|---|---|
| 客户需求分析页 | 生成报告 | 主按钮 | 调用 `/requirement/summary` | input_path 和 output_path 不为空 | loading | 显示成功提示和输出路径 | 显示 message 和 error_code |
| 客户需求分析页 | 重置 | 次按钮 | 恢复默认表单值 | 无 | 默认 | 清空结果区 | 无 |
| 客户需求分析页 | 复制输出路径 | 文本按钮 | 复制 output_path | 已生成成功结果 | 成功 | 显示已复制提示 | 显示复制失败 |

### 3. 页面状态设计

| 页面名称 | 状态 | 触发条件 | 页面表现 | 用户可操作项 |
|---|---|---|---|---|
| 客户需求分析页 | 默认 | 页面首次打开或点击重置 | 表单显示默认值，结果区为空 | 填写参数、生成报告 |
| 客户需求分析页 | 校验失败 | 必填字段为空时点击生成 | 字段下方显示错误提示，结果区不变化 | 修改字段 |

| 客户需求分析页 | 失败 | API 返回 success=false | 红色错误提示，展示 message 和 error_code | 修改参数、重试 |
| 客户需求分析页 | 网络异常 | 请求未到达后端或后端无响应 | 显示“后端服务不可用，请检查 FastAPI 是否启动” | 检查服务、重试 |

| 客户需求分析页 | output_path | 不能为空 | 输出目录存在或可创建 | 请输入报告输出路径 |
| 客户需求分析页 | enable_ai_advice | 无 | 开启时需要 DeepSeek API Key | 如果启用 AI 建议，请确认后端已配置 API Key |

## 十二、页面交互设计质量检查

完成后，用下面清单检查。

| 检查项 | 合格标准 |
|---|---|
| 字段是否完整 | 核心页面所需输入全部明确 |
| 字段标识是否清晰 | 字段标识能直接用于前端变量和 API 参数 |
| 按钮动作是否明确 | 每个按钮都知道点击后发生什么 |
| 校验是否明确 | 必填字段和错误提示清楚 |
| 状态是否完整 | 核心页面有默认、校验失败、loading、成功、失败 |
| 响应适配是否明确 | success/message/data/error_code 都有展示规则 |
| 是否避免过度设计 | 没有在一期加入复杂上传、权限、历史任务等能力 |
| 是否适合生成代码 | 可以直接作为 Vue3 页面 Prompt 的输入 |
