# 2-17 功能模块设计模拟工作台

## 本节目标

本节在 2-16 的“场景识别模拟工作台”基础上，继续完成原型生成流程的第三步：功能模块设计。

当前仍然不调用 DeepSeek，也不调用 FastAPI。本节通过前端模拟数据，把业务场景进一步收敛为功能模块、功能点、优先级和页面/API 映射。

## 本节价值

场景识别回答的是“谁在什么场景下完成什么任务”。

功能模块设计回答的是“为了支撑这些场景，一期原型应该做哪些功能，哪些功能可以后置”。

这一步可以帮助 FDE 在面对客户需求时控制范围，避免一开始就把所有功能都做进去。对于原型系统来说，最重要的是先抓住 P0 闭环，再逐步扩展 P1/P2。

## 页面设计

`FeatureDesignView.vue` 包含四类信息：

1. 优先级筛选
   - 全部
   - P0
   - P1
   - P2

2. 功能模块列表
   - 模块名称
   - 模块说明
   - 优先级
   - 来源场景

3. 当前模块详情
   - 功能点
   - 建议页面
   - API 方向

4. 范围控制说明
   - 一期必须做
   - 可以后置
   - 暂不进入原型

## 当前交互规则

- 页面默认展示全部功能模块。
- 点击优先级筛选按钮后，只展示对应优先级模块。
- 点击功能模块后，右侧展示该模块的功能点、页面建议和 API 方向。

## 后续扩展方向

后续可以继续扩展：

- 将 2-16 的场景识别结果传入功能设计页面
- 增加“确认 P0 范围”按钮
- 增加“生成页面清单”按钮
- 接入 DeepSeek 生成真实功能模块清单
- 将功能模块结果继续传给页面清单设计步骤

## 验收方式

启动前端后检查：

```bash
cd frontend
npm run dev
```

浏览器打开页面后确认：

- 工作区切换中出现“功能设计”
- 点击“功能设计”后进入功能模块设计模拟工作台
- 可以看到功能模块列表
- 点击 P0/P1/P2 筛选按钮后，模块列表会变化
- 点击不同模块，右侧详情会变化
- 第二阶段练习进度更新为 `2-17：功能模块设计模拟工作台`
