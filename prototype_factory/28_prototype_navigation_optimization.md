# 2-28 原型页面统一导航优化

## 本节目标

本节在已经完成 2-23 到 2-27 的 P0 业务原型页面基础上，为业务原型页面增加统一的内部导航区。

当前仍然不调用 DeepSeek，也不调用 FastAPI，不引入新的 npm 依赖。本节只优化前台页面之间的切换体验，让项目总览、计划任务、异常闭环、现场调度、物料到货五个 P0 页面可以在业务原型区内快速切换。

## 本节价值

现有工作区切换已经覆盖完整训练流程，包括需求输入、场景识别、功能设计、页面设计、API 契约和各个原型页。

但进入 P0 业务原型页后，用户还需要一个更聚焦的业务内部导航，用于模拟真实系统中的业务页面切换：

- 项目经理查看项目总览
- 计划员进入计划任务
- 现场调度员查看异常闭环和现场调度
- 物资协调员查看物料到货

这一步可以让当前前端原型更像一个连续可演示的业务系统。

## 导航设计

`PrototypeNav.vue` 作为通用组件，接收：

- `items`：业务原型导航项列表
- `activeKey`：当前激活的工作区 key

组件点击导航项后，通过 `select` 事件把目标 key 传回 `App.vue`。

当前导航项包括：

1. 项目总览
2. 计划任务
3. 异常闭环
4. 现场调度
5. 物料到货

## 当前交互规则

- 保留现有工作区切换按钮。
- 业务原型内部导航只在 P0 原型页面中展示。
- 点击内部导航后切换 `activeWorkspace`。
- 当前仍然只使用本地 mock 数据。
- 当前不引入路由，不引入新的 npm 依赖。

## 后续扩展方向

后续可以继续扩展：

- 将内部导航抽象为真实业务菜单。
- 增加页面分组和权限角色提示。
- 与后端项目列表、当前项目 ID、当前角色联动。
- 在接入 Vue Router 后，将 `activeWorkspace` 切换替换为路由跳转。

## 验收方式

启动前端或执行构建后检查：

```bash
cd frontend
npm run build
```

浏览器打开页面后确认：

- 原有工作区切换仍然存在。
- 点击“项目总览”“计划任务”“异常闭环”“现场调度”“物料到货”任一 P0 页面后，可以看到业务原型内部导航。
- 内部导航可以在五个 P0 页面之间切换。
- 非 P0 工作区不展示业务原型内部导航。
