# 客户需求结构化分析报告

## 一、业务目标

- 快速检索历史故障案例
- 自动归纳故障原因
- 生成标准化处置建议
- 沉淀专家经验

## 二、用户角色

- 现场工程师
- 设备管理员
- 专家审核人员
- 系统管理员

## 三、核心需求

- 支持导入故障案例
- 支持按设备类型检索
- 支持生成处理建议
- 支持专家审核和发布

## 四、初步功能模块建议

| 模块 | 建议 |
|---|---|
| 知识导入模块 | 支持故障案例、手册、专家经验等资料录入 |
| 知识检索模块 | 支持按设备类型、故障现象、关键词进行查询 |
| 智能建议模块 | 根据故障现象生成标准化处置建议 |
| 专家审核模块 | 支持专家复核、发布、版本留痕和回滚 |

## 五、初步分析结论

该需求具备典型的“知识沉淀 + 智能检索 + 专家审核 + 标准处置建议”特征，适合先建设一个轻量级故障知识助手 MVP。第一阶段不宜追求大而全，建议优先实现资料导入、基础检索、处置建议生成和专家审核闭环。

## 六、AI 辅助实施建议

### 1. 建设重点判断
- **核心价值在知识沉淀与复用**：项目成败关键在于能否将专家经验系统化、结构化地沉淀为可检索、可复用的故障知识库，而非简单的案例存储。建议优先定义故障案例的标准化模板（包含设备类型、故障现象、原因、处置步骤、审核状态等字段）。
- **快速检索能力是工程效率的底座**：现场工程师最关注“一秒找到类似案例”，因此需优先建设高性能的全文检索与多维度筛选（设备类型、故障代码、时间范围），并支持模糊匹配和标签联想。
- **闭环审核机制保障知识质量**：专家审核与发布流程是避免“垃圾进、垃圾出”的关键，必须设计明确的审批链路和版本管理，确保发布的处置建议经过验证、可追溯。

### 2. MVP 优先级建议
- **优先级高**：
  - 故障案例导入功能（支持 Excel/CSV 批量导入，字段映射校验）
  - 按设备类型、故障现象、关键字的多维度检索
  - 基础案例详情展示与处置建议查看
  - 专家审核流程（待审核→审核通过/驳回→发布，包含审批人节点配置）
  - 案例历史版本记录与回滚
- **优先级中**：
  - 自动归纳故障原因（基于 NLP 或规则引擎，初期可先预置常见原因模板，逐步引入机器学习）
  - 生成标准化处置建议（从历史成功案例中自动匹配最相似的处置步骤）
  - 用户权限管理（现场工程师、设备管理员、专家审核人员、系统管理员四类角色）
- **优先级低**：
  - 知识图谱关联分析（故障原因与设备、备件、工单的深度关联）
  - 移动端适配与离线使用
  - 与现有工单系统、EAM（企业资产管理）系统的接口集成

### 3. 主要风险点
- **案例数据质量参差不齐**：历史故障记录可能缺失关键字段（如设备编码、故障根因）、描述不规范，导致检索准确率低、自动归纳效果差。需在导入阶段强制校验必要字段，并提供清洗工具。
- **专家审核参与度不足**：专家日常工作繁忙，若无明确激励机制或流程约束，审核可能积压，导致新案例无法及时发布，降低系统时效性。建议设定 SLA 考核（如 24 小时内必审）。
- **自动归纳结果可信度低**：初期基于规则或简单模型的归纳可能产生歧义或错误，现场工程师若过度依赖会引发风险。应保留人工修正入口，并在建议中标注“置信度”或“来源案例编号”。
- **跨系统集成复杂度低估**：若需对接工单、资产台账系统，数据格式、API 接口稳定性、同步频率等细节容易引发返工。建议在 MVP 阶段优先独立运行，后期逐步集成。
- **用户习惯转变阻力**：现场工程师习惯口头交流或纸质记录，对新系统可能抵触。需提前准备培训材料与简单操作手册，并在试点阶段安排“种子用户”带动。

### 4. 乙方实施建议
- **需求澄清**：在项目启动前，必须与客户共同确认故障案例的“元数据标准”（至少包含：设备类型、故障日期、现象描述、原因分析、处置步骤、处置人、审核人、发布状态）。避免后期因字段缺失导致检索无效。
- **范围控制**：明确 MVP 仅覆盖“案例导入、检索、查看、审核发布”四步功能，自动归纳与建议生成列为第二阶段。合同中需注明“自动归纳准确率不做承诺，仅作为辅助参考”，避免验收争议。
- **数据准备**：要求客户在项目启动前提供至少 200 条历史故障案例（脱敏后），并安排业务骨干协同标注“标准原因”和“标准处置步骤”。同步提供数据清洗计划，确保导入后即能达到可用水平。
- **接口集成**：若客户要求与现有系统对接，需在合同中明确接口方式（RESTful API/数据库视图）、数据同步方向（单向/双向）、同步频率（实时/定时）、失败处理机制（告警+重试）。建议第一阶段仅做单向读取设备清单，避免写入依赖。
- **验收标准**：以可量化的指标设定验收条件，例如：
  - 检索响应时间：按任意关键词检索≤2秒（数据量≤10万条）；
  - 导入成功率：批量导入 1000 条案例，成功率≥99%；
  - 审核流程：完成从待审核到发布的全链路操作，耗时≤5 分钟（角色切换）；
  - 用户可用性：现场工程师完成一次从检索到查看处置建议的操作，平均耗时≤30 秒。
