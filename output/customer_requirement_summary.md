# 客户需求结构化分析报告

## 一、业务目标

- 快速检索历史故障案例
- 自动归纳故障原因
- 生成标准化处置建议
- 沉淀专家经验

## 二、用户角色

- 现场工程师
- 设备管理员
- 专家审核人员
- 系统管理员

## 三、核心需求

- 支持导入故障案例
- 支持按设备类型检索
- 支持生成处理建议
- 支持专家审核和发布

## 四、初步功能模块建议

| 模块 | 建议 |
|---|---|
| 知识导入模块 | 支持故障案例、手册、专家经验等资料录入 |
| 知识检索模块 | 支持按设备类型、故障现象、关键词进行查询 |
| 智能建议模块 | 根据故障现象生成标准化处置建议 |
| 专家审核模块 | 支持专家复核、发布、版本留痕和回滚 |

## 五、初步分析结论

该需求具备典型的“知识沉淀 + 智能检索 + 专家审核 + 标准处置建议”特征，适合先建设一个轻量级故障知识助手 MVP。第一阶段不宜追求大而全，建议优先实现资料导入、基础检索、处置建议生成和专家审核闭环。

## 六、AI 辅助实施建议

### 1. 建设重点判断

- **数据标准化与质量治理优先**：故障案例的结构化程度直接影响检索效率与自动归纳的准确性。必须优先定义统一的案例字段模板（设备类型、故障现象、根因、处置步骤、适用工况等），并建立历史案例清洗与补录机制，避免“垃圾进垃圾出”。
- **检索与知识图谱索引设计**：按设备类型检索是核心场景。应构建多维标签体系（设备型号、故障代码、工况参数、发生频次等），并支持模糊匹配与同义词扩展，确保现场工程师在短时间内精准命中目标案例。
- **专家审核-发布闭环流程**：生成处置建议后，必须设计轻量化的审核节点（如三级审批：初审-技术复核-最终发布），同时保留版本历史与回滚能力，防止错误建议上线引发次生风险。

### 2. MVP 优先级建议

| 优先级 | 建设内容 | 说明 |
|--------|----------|------|
| **高** | 案例批量导入与字段映射 | 支持Excel/CSV/API导入，含数据校验与重复检测 |
| **高** | 按设备类型+关键词组合检索 | 建立设备类型树状目录，支持全文检索与标签过滤 |
| **高** | 基于模板的标准化处置建议生成 | 根据案例库中高频处置步骤自动填充，人工可微调 |
| **高** | 专家审核工作台与发布管理 | 审核任务分配、批注、版本控制、一键发布/下线 |
| **中** | 自动归纳故障原因（规则引擎版本） | 基于预设规则（如“故障现象A+设备型号B→根因C”）而非纯AI，降低初期不确定性 |
| **中** | 用户角色与权限体系 | 区分现场工程师（只读+提交建议）、设备管理员（数据管理）、专家（审核）、系统管理员 |
| **低** | 趋势分析与案例统计看板 | 设备故障频率、案例采纳率等报表，供管理层决策参考 |

### 3. 主要风险点

1. **历史案例数据质量参差不齐**：客户提供的过往记录可能缺失关键字段（如根因、处置耗时），导致自动归纳准确率低，后续需投入大量人力补录。
2. **自动归纳结果不可靠引发信任危机**：若初期直接上线AI归纳，工程师因结果偏差而弃用整个系统，造成项目失败。需先以规则+模板兜底，渐进引入ML。
3. **专家审核流程阻塞**：审核人员日常工作繁忙，若无明确的超时自动流转或定期工作量统计，容易出现积压，影响新案例及时上线。
4. **与现有IT系统的集成耦合**：如设备管理系统的设备编码不规范、未对接统一认证，将大幅增加接口开发与数据同步成本。
5. **用户使用习惯迁移阻力**：现场工程师习惯个人笔记本或电话求助，若系统操作路径过长（需多步点击才能定位案例），会被视为效率负担。

### 4. 乙方实施建议

- **需求澄清**：明确“自动归纳故障原因”的业务边界——初期仅支持对结构化字段（如故障代码、现象关键词）的匹配映射，不承诺从自由文本中精准提取因果。需与客户共同定义至少20个典型故障的规则模板作为验收基线。
- **范围控制**：将“支持导入故障案例”限定为标准格式（如预定义模板），不接受临时格式的自动化解析。MVP阶段仅纳入前3种高频设备类型的案例，后续分期扩展。超范围需求（如移动端离线支持、语音输入）列入V2.0。
- **数据准备**：制定《案例数据规范手册》，包含必填字段（设备类型、故障现象、原因、处置步骤、适用条件）与校验规则。安排客户方数据专人在上线前完成至少500条历史案例的清洗录入，乙方提供数据转换脚本与质量报告。
- **接口集成**：优先对接客户现有的设备台账系统（获取设备型号、编码、运行参数），采用REST API + 消息队列的异步方式，避免关键业务链路堵塞。统一认证建议采用OAuth2.0或LDAP，减少多系统登录困扰。
- **验收标准**：
  - 功能验收：案例导入成功率≥99%（符合格式标准）；按设备类型检索平均响应时间≤2秒；生成处置建议的准确率≥85%（由专家评判）。
  - 非功能验收：单机支持并发50用户，7×24小时持续运行无崩溃；专家审核流程从提交到发布的平均时长≤2个工作日（按8小时工作制）。
  - 用户验收：组织10名现场工程师完成30个典型故障场景的模拟测试，全部成功找到合适案例并生成建议。
