# 海工制造 APS 原型系统设计建议

## 1. 原型系统总体定位

本原型系统定位为：

> **海工制造 APS 高级计划排程原型系统**

副标题建议：

> **面向导管架、管线、舾装、涂装、总装的多专业协同排程工作台**

该系统不是直接实现真实 APS 排程求解器，而是通过 **Mock 数据 + 可运行前端 + 甘特图 + 模拟排程接口**，直观表达海工制造 APS 的核心业务逻辑。

核心表达链路为：

```text
项目导入
→ WBS / BOM 拆解
→ 工艺路线建模
→ 资源能力建模
→ 物料齐套检查
→ 中期负荷计划
→ 短期甘特排程
→ 工作指令下发
→ 现场实绩反馈
→ 异常模拟与动态重排
```

该原型借鉴 Asprova APS 中 **中期计划、MRP、FCR、FCS、短期计划、工作指示、实绩反馈** 的核心思想，但面向海工制造场景进行行业化表达。

---

## 2. 原型面向的用户对象

本原型面向四类用户：

| 用户对象 | 重点诉求 | 原型中重点展示 |
|---|---|---|
| 客户领导 | 项目能否按期交付，风险在哪里 | 项目驾驶舱、交付风险、资源瓶颈、专业进度 |
| 计划员 | 如何排计划、看冲突、调资源 | 甘特图、中期负荷、短期排程、模拟重排 |
| 车间班组 | 今天干什么、在哪干、如何反馈 | 工作指令、班组任务、开完工反馈、异常上报 |
| 内部研发 | 数据如何组织，接口如何设计，组件如何拆 | Mock API、数据结构、页面组件、交互流程 |

因此，该原型不应只做领导大屏，也不应只做车间任务列表，而应设计为一个 **多角色 APS 工作台**。

---

## 3. 原型系统范围

本原型覆盖以下五类海工制造对象：

```text
导管架
管线
舾装
涂装
总装
```

建议第一版采用：

> **一个主案例 + 四个专业延展场景**

主案例建议使用 **导管架 JKT-001**，其他专业作为协同作业包纳入排程。

```text
主案例：导管架 JKT-001
├─ 导管架主结构：腿柱、斜撑、节点、片体
├─ 管线：管段预制、焊接、试压、安装
├─ 舾装：平台、栏杆、梯子、支架、阳极
├─ 涂装：喷砂、底漆、中间漆、面漆、修补
└─ 总装：片体合拢、吊装、定位、总装焊接、出运准备
```

这种设计既能聚焦导管架主线，又能体现海工制造多专业协同的复杂性。

## 4. 系统菜单设计

建议左侧菜单如下：

```text
1. 项目驾驶舱
2. 项目计划 / WBS
3. 制造 BOM / 施工包
4. 工艺路线
5. 资源能力
6. 物料齐套
7. 中期负荷计划
8. 短期甘特排程
9. 工作指令
10. 现场实绩
11. 异常与模拟重排
12. Mock 数据 / 接口说明
```

第一版最关键的页面：

```text
项目驾驶舱
中期负荷计划
短期甘特排程
物料齐套
工作指令
异常重排
```

这些页面能够完整表达 APS 的核心价值。

# 5. 页面级设计方案

## 5.1 项目驾驶舱

### 面向用户

客户领导、项目经理、售前演示人员。

### 页面目标

一眼看清项目总体进度、交付风险、资源瓶颈和物料齐套状态。

### 建议展示内容

| 区域 | 内容 |
|---|---|
| 顶部 KPI | 项目进度、计划完成率、交付风险、齐套率、资源负荷率 |
| 项目里程碑 | 设计冻结、主材到货、片体完成、总装完成、涂装完成、装船 |
| 风险卡片 | 主材晚到、NDT 超负荷、涂装房冲突、总装场地紧张 |
| 专业进度 | 导管架、管线、舾装、涂装、总装各专业进度 |
| 瓶颈资源 | 高级焊工、NDT、涂装房、总装场地、吊装资源 |
| 最近异常 | 返修、缺料、延期、设备故障 |

### 示例 KPI

```text
项目总进度：62%
计划完成率：87%
物料齐套率：76%
高风险任务：12 个
瓶颈资源：5 类
预计交付偏差：+3 天
```

## 5.4 工艺路线页面

### 面向用户

计划员、工艺人员、内部研发。

### 页面目标

展示不同专业对象的工艺路线。

### 导管架主结构路线

```text
材料到货 → 套料 → 切割 → 坡口 → 组对 → 焊接 → NDT → 打磨 → 涂装 → 入库 / 安装
```

### 管线路线

```text
管材到货 → 下料 → 坡口 → 组对 → 焊接 → NDT → 试压 → 涂装 → 安装
```

### 舾装路线

```text
材料到货 → 下料 → 预制 → 打磨 → 涂装 → 配送 → 安装 → 检查
```

### 涂装路线

```text
表面处理 → 喷砂 → 底漆 → 中间漆 → 面漆 → 漆膜检测 → 固化
```

### 总装路线

```text
场地准备 → 片体转运 → 吊装定位 → 组对 → 总装焊接 → NDT → 补漆 → 出运准备
```

### 工序详情字段

| 字段 | 示例 |
|---|---|
| 工序名称 | 焊接 |
| 前序 | 组对 |
| 后序 | NDT |
| 资源类型 | 高级焊工、焊机、焊接工位 |
| 标准工时 | 0.8 小时 / 焊口 |
| 是否质检点 | 是 |
| 是否允许外协 | 部分允许 |
| 齐套要求 | 图纸、焊材、WPS、工位 |

## 5.6 物料齐套页面

### 面向用户

计划员、采购员、车间班组。

### 页面目标

判断任务能不能开工、缺什么、影响什么。

### 页面结构建议

```text
左侧：施工包 / 构件列表
中间：齐套状态
右侧：缺料影响分析
```

### 齐套状态

```text
齐套
部分齐套
缺关键件
风险齐套
图纸未冻结
```

### 示例数据

| 任务 | 专业 | 齐套状态 | 缺失项 | 影响 |
|---|---|---|---|---|
| BRACE-001 组对 | 导管架 | 齐套 | 无 | 可开工 |
| PIPE-001 焊接 | 管线 | 部分齐套 | 阀门未到 | 试压延期 |
| OUTFIT-001 安装 | 舾装 | 风险齐套 | 阳极未到 | 不影响当前 |
| PAINT-001 面漆 | 涂装 | 缺关键件 | 面漆未到 | 涂装延期 |
| BLOCK-ASSY-001 | 总装 | 图纸未冻结 | Rev.C 未发布 | 禁止下发 |

物料齐套页面是海工 APS 原型中非常关键的页面。它比单纯甘特图更能体现海工制造的专业性。

## 5.7 中期负荷计划页面

### 面向用户

客户领导、计划员、项目经理。

### 页面目标

展示未来 3～6 个月资源瓶颈。

### 展示形式建议

```text
资源负荷柱状图 + 资源负荷表 + 系统建议
```

### 维度切换

```text
按专业：导管架 / 管线 / 舾装 / 涂装 / 总装
按资源：焊工 / NDT / 涂装房 / 总装场地
按周期：周 / 月
按项目：JKT-001 / 其他项目
```

### 示例负荷数据

| 资源 | 10 月 | 11 月 | 12 月 | 判断 |
|---|---:|---:|---:|---|
| 切割 | 76% | 83% | 71% | 正常 |
| 焊接 | 112% | 135% | 128% | 超负荷 |
| NDT | 96% | 148% | 132% | 严重瓶颈 |
| 涂装 | 62% | 88% | 156% | 后期瓶颈 |
| 总装场地 | 40% | 95% | 142% | 场地冲突 |

### 模拟建议

```text
1. 将 12 根斜撑转外协加工；
2. 11 月高级焊工增加 2 小时加班；
3. NDT 增加第三方检测资源；
4. 涂装任务提前分批释放；
5. 总装场地提前清场。
```

## 5.8 短期甘特排程页面

### 面向用户

计划员、车间主任、客户演示人员。

### 页面目标

用甘特图直观展示未来 7～35 天的详细排程。

这是原型系统的核心页面。

### 甘特图视图

建议支持四种视图：

```text
1. 按资源甘特图
2. 按专业甘特图
3. 按施工包甘特图
4. 按项目里程碑甘特图
```

### 甘特图任务示例

| 任务 | 专业 | 资源 | 开始 | 结束 | 状态 |
|---|---|---|---|---|---|
| LEG-01-S01 切割 | 导管架 | 切割机1 | 10-08 | 10-09 | 已排程 |
| BRACE-001 组对 | 导管架 | 铆工1班 | 10-09 | 10-10 | 待下发 |
| PIPE-001 焊接 | 管线 | 焊工2班 | 10-10 | 10-11 | 齐套风险 |
| OUTFIT-001 安装 | 舾装 | 舾装班组 | 10-12 | 10-14 | 待排 |
| PAINT-001 底漆 | 涂装 | 涂装房 | 10-15 | 10-16 | 资源冲突 |
| BLOCK-001 吊装 | 总装 | 履带吊 | 10-18 | 10-18 | 强约束 |

### 甘特图颜色建议

```text
蓝色：已排程
绿色：执行中
灰色：已完成
黄色：齐套风险
红色：资源冲突
紫色：关键路径
橙色：外协任务
```

### 交互建议

```text
点击任务：查看任务详情
点击运行排程：调用 mock 接口返回新排程
点击模拟异常：制造一个延期 / 缺料 / 资源冲突
点击重排：展示模拟后的新甘特图
```

## 5.11 异常与模拟重排页面

### 面向用户

计划员、客户演示人员、内部研发。

### 页面目标

展示 APS 如何处理变化。

### 预置模拟场景

```text
1. 大口径管材晚到 7 天
2. 高级焊工负荷 135%
3. NDT 检验排队
4. 涂装房资源冲突
5. 焊接实绩低于计划并产生返修
```

### 点击“模拟重排”后的 Mock 返回内容

```text
受影响任务
影响路径
延期天数
系统建议
调整后甘特图
```

### 示例

```text
异常：NDT 检验排队
影响：PIPE-001、BRACE-003、NODE-002 后续涂装延期
延期：预计 2 天
建议：增加第三方 NDT 资源；将非关键舾装任务后移；优先检测关键路径焊口。
```

# 9. 第一版原型演示主线

建议设计一条完整的演示故事：

```text
1. 客户领导进入项目驾驶舱
   → 看到 JKT-001 项目整体进度和交付风险

2. 计划员进入 WBS / BOM
   → 看到导管架、管线、舾装、涂装、总装如何拆解

3. 计划员运行中期负荷计划
   → 系统显示 NDT、高级焊工、涂装房超负荷

4. 系统给出模拟建议
   → 加班、外协、提前涂装、场地清理

5. 计划员进入短期甘特排程
   → 看到未来 35 天任务安排

6. 车间班组查看工作指令
   → 看到今天要做 BRACE-001 组对、PIPE-001 焊接

7. 现场反馈异常
   → 实际焊接只完成 52/80 道口

8. 系统模拟重排
   → 插入返修任务，调整 NDT 和涂装计划，提示延期 2 天
```

这条演示线既适合客户交流，也适合内部研发理解系统闭环。

# 10. 关键设计建议

## 10.1 不要做真实排程求解器

第一版原型不需要做复杂算法。应通过 Mock 数据模拟表达以下能力：

```text
运行中期排程
运行短期排程
模拟异常
模拟重排
生成工作指令
反馈现场实绩
```

这样更利于快速形成可演示系统。

## 10.2 必须有甘特图

APS 如果没有甘特图，就很难让客户直观理解排程结果。建议至少实现：

```text
资源甘特图
专业甘特图
施工包甘特图
```

第一版可以用自定义组件或 ECharts 时间轴模拟，不必追求完整商用甘特图能力。

## 10.3 物料齐套要做成核心页面

海工制造现场最常见的问题不是没有计划，而是：

```text
图纸未冻结
材料没到
场地没空
NDT 排不上
涂装房冲突
吊装窗口不可用
```

所以物料齐套页面要作为核心亮点来设计。

## 10.4 中期计划和短期计划要分开

中期计划回答：

```text
未来哪里会堵？
哪些资源会超负荷？
哪些物料要提前采购？
是否需要外协或加班？
```

短期计划回答：

```text
今天谁干？
在哪干？
干哪个构件？
需要什么物料？
干完怎么反馈？
```

这两个层级不能混在一起。

# 11. 建议的第一版数据量

为了保证原型既真实又不复杂，建议第一版 Mock 数据规模如下：

| 数据对象 | 建议数量 |
|---|---:|
| 项目 | 1 个主项目 |
| 专业 | 5 个：导管架、管线、舾装、涂装、总装 |
| WBS 节点 | 20～40 个 |
| 构件 / 施工包 | 40～80 个 |
| 工序任务 | 80～150 条 |
| 资源 | 20～40 个 |
| 物料 | 30～60 条 |
| 风险事件 | 10～20 条 |
| 工作指令 | 20～50 条 |

---

# 12. 最终建议

第一版原型建议直接按以下目标启动：

> **Vue3 可运行前端 + Mock 数据 + 甘特图 + 多角色业务闭环。**

第一版不追求真实排程算法，而是追求：

```text
业务逻辑讲得清
页面流程走得通
Mock 数据看起来真实
甘特图足够直观
齐套和风险能打动客户
研发能基于数据结构继续扩展
```

这个方向更适合快速做出可演示、可讨论、可迭代的海工 APS 原型系统。
